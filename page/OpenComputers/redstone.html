Description

This is the component provided by the Redstone card or block.

The tier one Redstone Card only supports vanilla redstone functionality (single-line analog redstone). Tier two provides interoperation with other mods' redstone systems (bundled, wireless). For bundled input/output we currently only support ProjectRed for Minecraft 1.12. In older versions we supported: RedLogic, Project Red (with Version 1.1), MineFactory Reloaded. For wireless redstone, the following mods are supported: WR-CBE, SlimeVoid's WR.

Component name: redstone.

Methods : Tier one

  • getInput(side: number): number returns the redstone level from the specified side.
  • getInput(): table returns a table of redstone levels from all sides.
  • getOutput(side: number): number Gets the currently set output on the specified side
  • getOutput(): table Get the currently set output of all sides
  • setOutput(side: number, value: number): number Sets the strength of the redstone signal to emit. Returns the old value.
  • setOutput(values: table): table Set the redstone strength of all sides.
  • getWakeThreshold():number Gets the current wake-up threshold.
  • setWakeThreshold(threshold:number):number Sets the wake-up threshold to the specified number.

Methods : Tier two

  • getBundledInput(side: number, color: number): number returns the strength of the incoming redstone value on the specified side on the specified color channel.
  • getBundledInput(side: number): table returns a table (Map[Int, Int] structure) of redstone values on the specified side in a bundled pack, indexed by color.
  • getBundledInput(): table returns all redstone values, of all sides and all colors. It is a Map[Int, Map[Int, Int]] structure. The top level keys are in [0, 5] range, values of sides (keep in mind sides.bottom is zero). The child map of each side is the same data structure returned by getBundledInput(side).
  • getBundledOutput(side: number, color: number): number returns the strength of the outcoming redstone value previously set that the device is emitting on the specified side on the specified color channel.
  • getBundledOutput(side: number): table returns a table (Map[Int, Int] structure) of redstone values previously set that the device is emitting on the specified side in a bundled pack, indexed by color.
  • getBundledOutput(): table returns all redstone values previously set that the device is emitting, of all sides and all colors. It is a Map[Int, Map[Int, Int]] structure. The top level keys are in [0, 5] range, values of sides (keep in mind sides.bottom is zero). The child map of each side is the same data structure returned by getBundledInput(side).
  • setBundledOutput(side: number, color: number, value: number): number set the value for the channel with the specified API/Colors. Returns the previous values set.
  • setBundledOutput(side: number, values: table): table sets a pack of color-indexed redstone values, [0, 15].The values table doesn't have to be contiguous, and values omitted are left unchanged.
  • setBundledOutput(values: table): table allows you to set redstone levels for any side and any color in a single api call.
  • getComparatorInput(side:number):number Get the comparator input on the specified side.
  • getWirelessInput():number Get the wireless redstone input.
  • getWirelessOutput():boolean Get the wireless redstone output.
  • setWirelessOutput(value:boolean):boolean Set the wireless redstone output.
  • getWirelessFrequency():number Get the currently set wireless redstone frequency.
  • setWirelessFrequency(frequency:number):number Set the wireless redstone frequency to use.
